前端命名规范

发表于 2020-01-15 | | 浏览 次
字数统计: 597

在一个团队开发中代码规范是很有必要的，有时候一个好的代码规范可以省略很多的备注，可以让自己或其他小伙伴很快地了解你是怎么样写的。。。

驼峰式命名

  • Pascal Case 大驼峰式命名法：首字母大写。eg：StudentInfo、UserInfo
  • Camel Case 小驼峰式命名法：首字母小写。eg：studentInfo、userInfo

文件资源名称

  • 文件名不含有空格
  • 文件名小写，不使用大写字母（除非：某些说明文件的文件名，可以使用大写字母，比如README、LICENSE )
  • 文件名包含多个单词时，单词之间使用半角的连词线 ( - ) 分隔

变量命名

命名方式：使用小驼峰命名

命名规范：类型+对象描述，如果没有明确的类型，就可以使前缀为名词

类型 小写字母
array a
boolean b
function fn
int i
object o
string s

推荐：

函数

命名方式： 小驼峰方式 ( 构造函数使用大驼峰命名法 )

命名规范： 前缀为动词

动词 含义
can 判断是否可执行某个动作 ( 权限 )
has 判断是否含有某个值
is 判断是否为某个值
get 获取某个值
set 设置某个值

推荐：

常量

命名方法 : 全部大写

命名规范 : 使用大写字母和下划线来组合命名，下划线用以分割单词。

推荐：

注释规范

单行
  • 单独一行：//(双斜线)与注释文字之间保留一个空格
  • 在代码后面添加注释：//(双斜线)与代码之间保留一个空格，并且//(双斜线)与注释文字之间保留一个空格。
  • 注释代码：//(双斜线)与代码之间保留一个空格。

推荐：

多行
  • 若开始(/和结束(/)都在一行，推荐采用单行注释
  • 若至少三行注释时，第一行为/，最后行为/，其他行以开始，并且注释文字与保留一个空格。
